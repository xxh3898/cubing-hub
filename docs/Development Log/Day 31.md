## 핵심 정리 상세

### 1. 초보자 해법은 별도 페이지가 아니라 학습 탭으로 붙였다

#### 선택

- 새 라우트를 만들지 않고 기존 `/learning` 화면에 `초보자` 탭을 추가했다.
- 탭 순서는 `회전기호 가이드`, `초보자`, `F2L`, `OLL`, `PLL`로 뒀다.
- 처음 사용자는 회전기호를 먼저 확인하고, 바로 다음 탭에서 초보자 8단계 목록을 볼 수 있다.

#### 이유

- 학습 기능은 이미 정적 데이터와 VisualCube 카드 패턴을 갖고 있다.
- 초보자 해법도 같은 화면 안에 두면 라우팅과 API 범위를 늘리지 않고 현재 사용자 흐름을 유지할 수 있다.

### 2. 초보자 단계는 단계 목록과 케이스 데이터를 분리했다

#### 구현

- 기존 CFOP 데이터의 `algorithm` 필드는 그대로 유지했다.
- `beginnerSteps`에는 1단계부터 8단계까지의 제목, 설명, 케이스 수 또는 안내 메타를 둔다.
- `beginnerStepCompletions`에는 각 단계를 끝냈을 때의 목표 상태를 둔다.
- `beginnerCases`에는 각 단계에서 보여줄 케이스와 해법을 둔다.
- `beginnerCases`에는 공식을 적용해야 하는 상태만 남기고, 이미 맞은 상태는 완료 모습으로만 보여준다.
- 같은 해법을 반복하는 케이스에는 `visualAlgorithm`에 반복 횟수나 setup 회전을 넣어 카드마다 다른 VisualCube 상태를 만든다.
- 1단계 `십자 맞추기`는 `caseCount`를 `0`으로 두고 단계 목록에서는 `안내`로 표시한다.
- 1단계 상세 화면은 흰색 센터를 위로 두고, 흰색 엣지를 빈 십자 자리로 옮기는 흐름만 보면 된다고 안내한다.
- 1단계 상세 화면은 케이스 카드 그리드를 렌더링하지 않고 흰색 십자 완료 모습만 보여준다.
- 1단계 상세 화면의 가이드와 완료 모습 이미지는 흰색 외 facelet을 회색으로 단순화해 옆면 색 정렬과 혼동하지 않게 했다. 이미지는 기본 3D 각도로 렌더링해 회전기호 가이드와 같은 시점으로 본다.
- 단계 상세 상단에는 `단계 목록`과 다음 단계가 있을 때 `다음 단계 (n단계: 제목)` 버튼을 먼저 보여주고, 그 아래에 현재 단계 제목과 설명을 표시한다. 마지막 8단계에서는 다음 단계 버튼을 숨긴다.
- 2단계 `십자 엣지 맞추기`는 완료 모습과 인접/반대 케이스를 `visualFaceletColors`로 직접 지정한다. 흰색 십자는 상단에 두고, 옆면 top edge와 center가 맞거나 어긋난 상태를 구분한다. 완료 모습과 케이스는 기본 3D 각도로 렌더링하고, 인접/반대 케이스는 `stage` 파라미터 없이 렌더링해 옆면 색이 회색으로 마스킹되지 않게 했다.
- 3단계 `1층 맞추기`는 케이스 1-3의 공식 기반 `case + stage=f2l` 그림만 유지한다. 케이스 4 `잘못 들어간 코너 빼내기`는 초보자 화면에서 제외한다.
- 4단계 `2층 맞추기`는 오른쪽/왼쪽 삽입 2개 케이스만 유지한다. 케이스 3 `잘못 들어간 엣지 빼내기`는 초보자 화면에서 제외한다.
- 4단계 완료 모습은 `visualFaceletColors`만 사용하고 `stage`를 비워 VisualCube 기본 3D 각도로 렌더링한다.
- 5단계 `윗면 십자 맞추기`는 `visualFaceletColors`를 사용해 윗면 노란색 패턴을 직접 지정한다. 케이스 1 `ㄱ자`는 윗면 위쪽과 왼쪽 엣지만 노란색으로 둔다.
- 6단계 `윗면 맞추기`는 `visualFaceletColors`를 사용해 윗면 십자는 유지하고 노란 코너 방향만 미완성 7가지로 나눈다. 각 코너 조각의 노란 스티커가 정확히 한 곳에만 있도록 옆면 코너 스티커까지 지정하고, 케이스별로 공식 `R' F' L' F R F' L F`와 필요한 `U`, `U'`, `U2` 기준 조정을 순서대로 표시한다.
- 7단계 `3층 코너 맞추기`는 `visualFaceletColors`를 사용해 인접/대각선 교환 상태를 직접 지정한다. 윗면은 모두 노란색으로 표시하고, 옆면의 3층 엣지만 회색으로 가리며, 1층과 2층의 맞춰진 색은 유지한다. 대각선 교환은 인접 교환 공식으로 해결되지 않아 전용 대각선 교환 공식을 사용한다.

#### 결과

- 기존 `F2L`, `OLL`, `PLL` 데이터는 대량 수정 없이 그대로 동작한다.
- `초보자` 탭은 단계 목록을 먼저 보여주고, 선택한 단계에서 완료 모습을 먼저 확인한 뒤 기존 CFOP 카드와 같은 방식으로 케이스를 탐색할 수 있다.
- VisualCube URL에는 공식 기반 `case` 또는 직접 지정한 `fc` facelet 색상만 들어간다.
- 각 초보자 단계 안에서 케이스 이미지 `src`가 중복되지 않는지 테스트로 확인한다.
- 1단계는 잡는 방향 안내와 완료 모습만 보이고 케이스 이미지, 공식 문구, 수순 코드 블록이 렌더링되지 않는지 테스트로 확인한다. 1단계 가이드와 완료 모습 URL에는 `view`가 없는지도 확인한다.
- 다음 단계 버튼으로 1단계에서 2단계로 이동할 수 있고, 8단계에서는 다음 단계 버튼이 없는지 테스트로 확인한다.
- 2단계 완료 모습과 인접/반대 케이스는 `case`가 아니라 `fc`를 쓰고, 흰색 십자가 상단에 있으며 두 케이스 그림이 서로 다른지 테스트로 확인한다. 완료 모습과 인접/반대 케이스 URL에는 `view`가 없고, 인접/반대 케이스 URL에는 `stage`가 없는지도 함께 확인한다.
- 3단계는 기존 `case + stage=f2l` URL을 쓰는 케이스 1-3만 렌더링하고, 케이스 4가 렌더링되지 않는지 테스트로 확인한다.
- 4단계는 기존 `case + stage=f2l` URL을 쓰는 오른쪽/왼쪽 삽입 케이스만 렌더링하고, 케이스 3이 렌더링되지 않는지 테스트로 확인한다.
- 4단계 완료 모습 URL에 `stage`와 `view`가 없는지 테스트로 확인한다.
- 6단계 7개 케이스 URL이 모두 `fc`를 쓰고, 완성 상태를 제외한 미완성 7개인지, 각 코너 조각의 노란 스티커가 윗면 또는 인접 옆면 중 한 곳에만 있는지, 표시된 공식 순서를 적용하면 윗면이 완성되는지 테스트로 확인한다.
- 7단계는 완료/인접/대각선 그림이 모두 `fc`를 쓰고, 윗면은 모두 노란색이며, 옆면 3층 엣지 조각만 회색 처리되고, 1층과 2층 색이 유지되며, 화살표 오버레이 없이 렌더링되는지 테스트와 스크린샷으로 확인한다. 인접/대각선 케이스는 표시 공식을 실제 facelet에 적용했을 때 완료 facelet으로 도달하는지도 테스트로 확인한다.
- 5단계 `케이스 1`은 노란 코너가 보이지 않고, `케이스 3`은 중앙 노란색만 있는 점 상태로 렌더링된다.
- 모든 초보자 단계는 완료 모습 카드 1개를 먼저 보여준다.
- 2단계, 7단계, 8단계의 이미 맞은 상태는 케이스 목록에서 제거해 케이스 이미지가 공식 적용 전 상태만 나타내도록 했다.

## 남은 리스크

- 초보자 케이스별 VisualCube 이미지는 그림용 상태값과 일부 facelet 색상으로 구분한다. 다만 블로그 원본 그림을 복제한 전용 다이어그램은 아니다.

- worktree에서 `npm ci`를 실행했을 때 `npm audit` 기준 moderate 취약점 2건이 보고됐다. 이번 변경에서 의존성을 추가하거나 갱신하지는 않았다.
